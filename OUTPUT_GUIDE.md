## Running the Simulation

### Command
```bash
cd C:\Users\Aarya\Campus-network-evaluation
python run_enhanced_simulation.py
```

### What Happens
1. Loads `config.yaml` (network parameters)
2. Runs **Normal Mode** simulation (60 seconds)
   - All traffic destinations allowed
   - ~429 packets generated
3. Runs **Exam Mode** simulation (60 seconds)
   - Only EXAM_SERVER and EMAIL_SERVER allowed
   - ACL blocks 50% of traffic
   - ~430 packets generated
4. Generates all outputs to `./output/` directory
5. Prints status messages to console

### Output Timeline

## Visualizations (PNG Files)

### 1. VLAN Traffic Volume
**Files**: `vlan_traffic_normal.png`, `vlan_traffic_exam.png`

**What It Shows**:
- Stacked bar chart with delivered (green) and lost (red) packets per VLAN
- X-axis: VLAN IDs (101-108 main campus, 201-204 campus2, 128 servers)
- Y-axis: Packet count
- Shows which VLANs have higher traffic and loss rates

**What Makes It**:
- **Module**: `visualization_enhanced.py` line 40-84
- **Data Source**: `NetworkAnalyzer.get_vlan_statistics()`
- **Generated By**: `run_enhanced_simulation.py` line 262 (normal), line 316 (exam)
- **Uses**: Pandas DataFrame aggregation by VLAN

**Example Interpretation**:
- VLAN 101 has 54 packets, 50 delivered (92.6%), 4 lost
- VLAN 128 (servers) has highest volume (429 total)
- Loss rates consistent across VLANs (~8% in normal mode)

## Data Exports (CSV/JSON)

### 1. Packet-Level CSV
**Files**: `simulation_results_normal.csv`, `simulation_results_exam.csv`

**What It Contains**:
- **Format**: CSV with 14 columns
- **Rows**: One row per packet (429 normal, 430 exam)
- **Header**: `timestamp,src_ip,dst_ip,src_name,dst_name,src_type,dst_type,src_vlan,dst_vlan,bytes,delay_ms,hops,status,loss_reason`

**Column Definitions**:
| Column | Example | Meaning |
|--------|---------|---------|
| timestamp | 0.0 | Seconds in simulation (0-60) |
| src_ip | 10.1.101.50 | Source IP address |
| dst_ip | 10.128.0.10 | Destination IP address |
| src_name | Client_0 | Source device friendly name |
| dst_name | EXAM_SERVER | Destination device name |
| src_type | pc | Device type (pc, server, printer, etc) |
| dst_type | server | Destination device type |
| src_vlan | 101 | Source VLAN |
| dst_vlan | 128 | Destination VLAN |
| bytes | 1500 | Packet size in bytes |
| delay_ms | 4.27 | End-to-end delay in milliseconds |
| hops | 4 | Number of hops traversed |
| status | delivered | "delivered" or "dropped" |
| loss_reason | none | "none", "link_loss", "acl_blocked", "ttl_exceeded" |

**What Makes It**:
- **Module**: `data_exporter.py` line 65-110
- **Data Source**: `PacketCollector` list (created during simulation)
- **Generated By**: `run_enhanced_simulation.py` line 284 (normal), 325 (exam)
- **Uses**: Pandas DataFrame.to_csv() with packet attributes

**Example Row**:
```
0.0,10.1.0.0,10.128.0.10,Client_0,EXAM_SERVER,pc,server,101,128,1500,4.273,4,delivered,none
```

**Uses**:
- Excel/Tableau analysis
- Custom filtering and aggregation
- Per-device troubleshooting
- Loss reason investigation

## Data Structure Reference

### Packet Record (14 attributes)
Created by `PacketCollector` during simulation, tracked in dataframe:

```python
@dataclass
class PacketRecord:
    timestamp: float          # 0.0 - 60.0 seconds
    src_ip: str              # 10.1.101.50 format
    dst_ip: str              # 10.128.0.10 format
    src_name: str            # PC_MAIN_F1_1
    dst_name: str            # EXAM_SERVER
    src_type: str            # pc, server, printer, phone, laptop, ap
    dst_type: str            # server, pc, etc
    src_vlan: int            # 101-108, 201-204, 128
    dst_vlan: int            # 101-108, 201-204, 128
    bytes: int               # packet size (1500)
    delay_ms: float          # 1.34 - 9.64 milliseconds
    hops: int                # 2 - 8 hops
    status: str              # "delivered" or "dropped"
    loss_reason: str         # "none", "link_loss", "acl_blocked", "ttl_exceeded"
```

## Configuration File (config.yaml)

Controls simulation behavior without code changes:

```yaml
simulation:
  duration_seconds: 60
  packet_loss_rate: 0.02
  max_ttl: 64
  random_seed: 42

network:
  main_campus:
    floors: 8
    devices_per_floor: 12
  campus2:
    floors: 4
    devices_per_floor: 12

bandwidth:
  wan_mbps: 10000
  core_mbps: 8000
  distribution_mbps: 4000
  access_mbps: 1000
  end_device_mbps: 100

traffic:
  packet_generation_interval_ms: [100, 500]
  traffic_weights:
    exam_server: 0.40
    email_server: 0.20
    cloud_server: 0.10
    internet: 0.10
    peer_to_peer: 0.20

visualization:
  show_plots: true
  save_formats: ["png"]
  dpi: 300

output:
  output_directory: "./output"
  export_csv: true
  export_json: true
  export_pickle: false
  generate_text_report: true
  generate_comparison: true
```

## Common Use Cases

### 1. Verify Network Performance
1. Run simulation
2. Check `network_evaluation_report_*.txt` for findings
3. View `vlan_traffic_*.png` for per-VLAN analysis
4. View `delay_distribution_*.png` for latency

### 2. Compare Normal vs Exam Mode
1. View `simulation_comparison.json` for deltas
2. Compare `delivery_rate` (-29.5% in exam)
3. View `loss_breakdown_*.png` to see ACL impact
